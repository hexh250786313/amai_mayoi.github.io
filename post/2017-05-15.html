從零開始的建博過程

申請自己的個人域名和修改相關設置

强烈建議有一定HTML基礎的同學使用本教程，如果沒有任何基礎，這篇教程就此可以結束了（除非你真的很愛折騰...）。
首先需要租一個自己的域名，可以到各大出租域名的站點購買。我是在GoDaddy上買的，因爲支持支付寶。先注冊賬號，然後搜自己想要的域名租買即可，結賬的時候會提供各種附加服務，斟酌購買，不購買任何附加服務也是可以的，過程不表。
接著在賬戶中找到自己的新域名，找到該域名下的DNS管理，接著要把新域名的服務器綁定到騰訊旗下的DNSPod域名管理器下。點解域名服務器的更改按鈕，然後把兩個服務器地址修改成f1g1ns1.dnspod.net和f1g1ns2.dnspod.net，也就是騰訊的域名服務器。
然後一般來説等若干小時，域名服務器那裏提示被外部服務器接管則表示成功了。

DNS設置

接著就是DNS的相關設置，既然之前使用了DNSPod的域名管理器，那我們需要先到DNSPod中注冊賬號。
然後進入賬戶中新添加域名，也就是剛買的的新域名，接著進入域名管理界面。在這裏會看到很多的記錄，先把A類型的記錄全部暫停掉，否則A類型會不夠用（沒有則不用管）。然後新添加兩條A記錄，名字隨便起，記錄值設定為192.30.252.153和192.30.252.154，其他保持默認。（註：這兩個記錄值是GitHub Page的IP地址）

設置GitHub

因爲是要利用GitHub提供的免費的GitHub Page服務器，所以當然先需要一個GitHub賬號，先進去注冊。
完了以後會發現可以新建倉庫(repositories)，這裏的倉庫(repositories)的意思是指代碼倉庫，即程序員會把代碼交給Github進行云托管。而這裏不需要新建倉庫，而是直接Fork別人的博客倉庫。具體操作是在百度或Google搜索GitHub Page模板，找到好看的以後點進去他的GitHub中，點右上角的Fork。回到自己的主頁，會發現別人的博客代碼倉已經出現在了自己的倉庫中。
接著在Fork回來的博客倉中點擊設置(Settings)，頁面往下拉找到GitHub Page，在下拉欄中選擇Master Branch，然後在提示的輸入欄中填入自己的域名，保存即可。
至此，你在瀏覽器中輸入自己的域名進入，就會發現已經變成了Fork回來的模板的樣子了。然後就可以通過提交HTML文檔或者使用MarkDown文檔來提交你的日誌了。

關於博客如何更新

Git設置

如果通過GitHub的在綫編輯器修改編輯文檔會非常地麻煩，所以可以通過在計算機下載Git進行本地提交。不管是Windows還是Linux，先到Git的官方網站下載安裝Git到電腦上。
接著使用命令行輸入以下代碼檢查存放SSH密鑰的文件夾，Windows的話直接在文檔中檢查是否有名爲.ssh的文件夾。
cd ~/.ssh
如果提示沒有則新建一個，如下。
mkdir .ssh
接著在該文件夾下生成新的SSH密鑰。
ssh-keygen -t rsa -C "<邮件地址>"
提示輸入文件名時鍵入id_rsa，然後設定密碼即可。完成後會在.ssh文件夾下看到id_rsa和id_rsa.pub文件。然後用文本編輯器打開id_rsa.pub，複製其全部密鑰内容。
接著進入你的GitHub賬號，在賬戶設置中左側索引找到SSH and GPG keys，新添加一個記錄，名字隨意，把之前拷貝的内容粘貼上去。
在命令行中測試連接是否成功，如下。中途會要求鍵入YES，如果最終提示成功，則成功。
ssh -T git@github.com
提示成功后，則説明你的電腦已經可以利用Git管理GitHub的代碼了。完善下Git信息，如下。
git config --global user.name "<真实姓名>"
git config --global user.email "<邮箱地址>"

利用終端命令進行更新

首先利用Git把你的博客倉克隆到本地，如下。
git clone <你的博客倉地址>
然後你就可以編輯克隆到本地的代碼倉以編輯博客樣式和日誌文檔，接著可以利用如下代碼重新提交回GitHub。
git add --all
git commit -m "Initial commit"
git push -u origin master
提示輸入賬號的時候輸入你的GitHub賬號即可。
